<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (Manager) → FastAPI reminders routes | Untrusted JSON crosses here: schedule fields, message body, mentions, reactions, and a claimed `version`. |
| FastAPI panel ↔ scheduler (shared sqlite) | The panel is the second writer; version guard + server-side recompute are the integrity controls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-11 | Elevation of Privilege | any reminders route reachable without Manager tier | mitigate | `Depends(require_manager)` on every route (V4); gate test asserts 403 for non-manager |
| T-06-12 | Tampering | client-supplied next_fire_utc bypassing schedule validation | mitigate | Panel always recomputes next_fire_utc server-side from validated fields (V5); never reads it from the body for a write |
| T-06-13 | Tampering (lost update) | stale panel edit overwriting a newer row (REM-03 panel half) | mitigate | `expected_version` threaded through update/delete; False → 409 reload, no silent overwrite |
| T-06-14 | Tampering | version fabricated by client to force overwrite | mitigate | version only ever matched in WHERE + incremented server-side (Plan 02); route echoes back the last-read value only |
| T-06-15 | Injection | message/mention/reaction fields reaching the store | accept | Parameterized `?` placeholders everywhere (T-08-03); Jinja2 auto-escapes; body never rendered as HTML in the table (V5) |
| T-06-16 | Tampering (CSRF) | forged POST to create/edit/delete/pause/resume | mitigate | Existing `SameSite=Lax` session cookie is the app-wide CSRF control (T-10-10-04), same as Settings — no new token needed |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No packages installed this phase (RESEARCH.md audit N/A) |
</threat_model>
