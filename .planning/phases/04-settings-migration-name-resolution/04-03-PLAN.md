| Boundary | Description |
|----------|-------------|
| browser → FastAPI settings route | Untrusted form input crosses here; server-side allowlist validation (unchanged) is the boundary. |
| shared sqlite → app process | The app is a read-only consumer of discord_names; it never writes it and never imports discord. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing / Elevation of Privilege | app resolving Discord names | mitigate | The app resolves names ONLY from `db.get_discord_names` (sqlite); it never imports discord, never calls fetch_channel/fetch_guild, and holds no bot token. Grep gate: `grep -rn 'import discord\|fetch_channel\|BOT_TOKEN' app/` returns nothing. |
| T-04-02 | Tampering | snowflake precision shipped to the browser | mitigate | The names map is built with the TEXT `id` as string keys and shipped via `| tojson`; JSON object keys are strings, so ids > 2**53 never round (test_names_map_shipped_to_alpine_is_string_keyed). |
| T-04-03 | Elevation of Privilege | owner-only boundary / 403 handler drift | mitigate | `require_owner` and `save_settings` are byte-unchanged (D-03); the route stays `/admin/settings` so the 403 special-case still fires; the test_app_dashboard settings-403 assertions guard it. |
| T-04-04 | Tampering | cache poisoning of displayed names | mitigate | The app never writes discord_names; resolved names are display-only and are never fed back into `settings.set` (the POST body carries only raw ids validated by the unchanged allowlist). |
| T-04-05 | Denial of Service | cold/missing discord_names table → 500 | mitigate | `db.init_discord_names()` added to lifespan (dual-process defensive init); `_read_name_cache` tolerates an empty result → cold banner, never a 500. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs this phase. |
</threat_model>
