  <what-built>
    The v1 owner settings panel migrated into the dashboard shell as the Settings section
    (extends _dashboard_base, sidebar + topbar from the shell, one Save action), with
    channel/role fields resolving readable names from the bot-pushed discord_names cache:
    #channel with a type cue + raw ID beneath, one color-barred @role chip per role_list id,
    a "Name unavailable · no encontrado" marker for a genuinely-unknown id, and a section-level
    "Sincronizando nombres · Names syncing" banner when the cache is cold/stale. The two-pass
    validate-then-write POST, per-field errors, and the require_owner gate are unchanged.
  </what-built>
  <how-to-verify>
    Prerequisite: the bot process must be running at least once so the cache is populated
    (the push cog runs within ~5 min of startup; until then the cold banner is expected).

    1. Sign in as the owner and open the Settings section from the sidebar (⚙ "Ajustes · Settings").
       Expect the variant-A shell chrome (dark SaaS, Inter, sidebar with per-module accents,
       the shared topbar) — NOT the old bespoke settings topbar.
    2. Confirm each feature group renders as a card (Gallery, Reviews, Reminders, Jinxxy, Meetings,
       Forum, and Access LAST). Confirm there is exactly ONE Save action for the whole page.
    3. For a channel field (e.g. a gallery/reviews/forum channel id): confirm it shows a readable
       #name with a channel-type cue and the raw ID in mono beneath.
    4. For a role_list field (e.g. Access → manager_roles/editor_roles, or GALLERY_STAFF_ROLE_IDS):
       confirm one chip per role, each with a 3px left color bar tinted the role's actual Discord
       color, the @name, and the id in mono. An empty field shows "Sin roles asignados · No roles assigned".
    5. Paste a valid-but-nonexistent 18-digit id into a channel field: with a fresh cache, confirm
       the "Name unavailable · no encontrado" marker appears; the field still accepts and can save.
       Confirm the name preview updates live as you type/paste (before saving).
    6. Cold-cache check (optional): with the bot stopped/never-pushed, confirm ONE section banner
       "Sincronizando nombres · Names syncing" appears and fields show raw ids WITHOUT per-field
       "unavailable" markers.
    7. No-loss check: change one valid value and Save → success toast "Ajustes guardados. — Settings saved."
       Then submit a mix of one valid + one invalid value → 422, inline error on the bad field, and
       confirm the valid field was NOT written (reload and verify it kept its prior value).
    8. Access check: from a Manager (non-owner) session, click the locked Settings nav item →
       confirm the styled in-shell forbidden page (owner-tier dead end), not the editor login page.
  </how-to-verify>
