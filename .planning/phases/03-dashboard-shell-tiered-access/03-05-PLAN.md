<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser session → dependency gate | Identity from signed session only; tier facts from a live server-side Discord REST read — never from request body/query |
| app → Discord REST | Bot token only, timeout-bounded; role authority is always the bot-token read, never the OAuth token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-12 | Elevation of Privilege | Manager self-elevation | mitigate | Tier facts derive from live roles + owner-only mapping edit; require_owner (unchanged) gates the mapping save — a Manager session can never satisfy it |
| T-03-13 | Denial of Service | owner lockout via bad mapping | mitigate | is_owner is hardcoded to DISCORD_USER_ID before any role lookup; the mapping cannot express/remove ownership (D-04); fails closed when DISCORD_USER_ID unset |
| T-03-14 | Tampering / EoP | stale-session privilege retention | mitigate | Every protected request re-reads live guild roles (D-02); no tier is cached in the session — instant revocation on role removal |
| T-03-15 | Tampering (IDOR) | client-supplied identity/tier | mitigate | discord_id from session, all tier facts from server-side REST read; nothing tier-related read from request body/query |
| T-03-16 | Tampering | open redirect via post-login target | mitigate | Post-login path is a fixed server-computed internal route per tier, never a client ?next |
| T-03-17 | Rate-limit / DoS | N+1 Discord REST calls per page | mitigate | One _resolve_roles dependency shared via FastAPI use_cache — one REST read per request (Pitfall 4) |
| T-03-SC | Tampering | pip installs | accept | No new package (reuses httpx/authlib/starlette already installed) |
</threat_model>
