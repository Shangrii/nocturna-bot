# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Visual source of truth:** `.planning/sketches/001-dashboard-shell/index.html` (variant A —
"MEE6 puro", winner), `.planning/sketches/themes/default.css`, `.planning/sketches/MANIFEST.md`.
These are pre-validated via `/gsd:sketch` and treated as locked. This spec transcribes them into
an actionable contract and adds Phase-3-specific component/copy decisions the sketch didn't cover
(stub pages, 403 page, lock-icon nav, out-of-scope toggle removal).

## Spacing Scale

Declared values (must be multiples of 4), transcribed from `default.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, switch thumb inset, accent bars (`.acc-bar` width) |
| sm | 8px | Nav-item/brand internal gaps, button row gaps, chip gaps |
| md | 16px | Default field/card-header spacing, stats/gallery-queue grid gaps, sidebar padding |
| lg | 24px | Card padding, card margin-bottom, page-header (`.ph`) margin-bottom |
| xl | 32px | Main content area padding (`.main`), empty-state padding |
| 2xl | 48px | Reserved for major section breaks — not used by any Phase-3 component yet; keep available, do not invent a use |
| 3xl | 64px | Not used in the sketch or this phase — no page in Phase 3 needs a break this large |

Exceptions: **12px** (`--space-3`) — used for the sidebar group-label top padding (`.grp`) and the
gallery-card body inner padding (`.gcard .b`). This is a pre-existing value in the locked sketch
theme, not a new deviation; keep it as-is rather than rounding to 8 or 16. **This is the only
permitted exception** — no further ad-hoc spacing values (e.g. 10px, 14px, 20px) may be introduced
anywhere in Phase 3 markup/CSS; every other measurement must resolve to one of the scale tokens
above.

## Typography

Exactly 4 sizes, exactly 2 weights — closed set, no other `font-size`/`font-weight` value may
appear in Phase 3 markup or CSS.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px (`--text-xs`) | 600 semibold | 1.2 — uppercase labels, table headers, badges, hints, group headings |
| Body | 13px (`--text-sm`) | 400 regular | 1.5 — body element default, table cell text, nav items, chips, muted copy, review/quote text |
| Heading | 18px (`--text-lg`) | 600 semibold | 1.2 — module header title (`.mod-hdr .t`), Overview stat-tile numeric value (`.stat .v`) |
| Display | 26px (`--text-2xl`) | 600 semibold | 1.2 — page title (`.ph .ttl`), i.e. "Overview", "Recordatorios", etc. |

Font weights used: **400 regular** (body/muted text only) and **600 semibold** (everything else —
labels, buttons, card headers, badges, module titles, stat values, page titles). No 700/bold and
no 800 anywhere in Phase 3 (the sketch's `--font-sans` 700/800 usages — page title, stat value,
brand-logo monogram — all collapse onto 600 semibold for this phase's implementation). Load only
Inter 400 and 600 from Google Fonts; drop the 700 weight from the font `@import`/`<link>` list.

`--text-base` (15px) and `--text-xl` (21px) from `default.css` are **not used** anywhere in Phase 3
— every element that the sketch styled with those two tokens is re-mapped onto Body (13px) or
Heading (18px) respectively per the table above, so the rendered UI only ever shows the 4 declared
sizes.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1115` (`--color-bg`) | Page background |
| Secondary (30%) | `#161a21` (`--color-surface`), `#1d222b` (`--color-surface-2`), `#252b36` (`--color-surface-3`) | Sidebar, cards, module headers, stat tiles, form inputs, badge/chip/pill backgrounds |
| Accent (10%) | `#4f7cff` (`--color-primary`) | Reserved for: primary buttons (`.btn`), active/selected sidebar nav-item left-border + hover, input focus ring, default stat-tile accent bar when no module color applies |
| Destructive | `#f0435a` (`--color-danger`) | Reserved for: destructive/reject buttons only (`.btn.danger`) — Phase 3 has no delete actions, but the token is declared for the forbidden-page tone-of-voice and future phases |

Accent reserved for: primary interactive controls listed above ONLY — never decorative fills,
never body text color, never card borders in general.

**Second accent system — per-module section colors (locked, distinct from the 10% interactive
accent above):** these are section-identity accents, not general UI accents. Reserved exclusively
for: sidebar nav-item left-border for that section, that module's header left-border + icon color,
and that module's stat-tile top-border on Overview.

| Section | Accent |
|---------|--------|
| Overview | `var(--color-primary)` `#4f7cff` |
| Gallery | `#b96bff` (`--accent-gallery`) |
| Reviews | `#ffb020` (`--accent-reviews`) |
| Reminders | `#38bdf8` (`--accent-reminders`) |
| Jinxxy Store | `#34d399` (`--accent-jinxxy`) |
| Meetings | `#f472b6` (`--accent-meetings`) |
| Settings | `#8b93a3` (`--accent-settings`) |

Semantic status colors (not decorative, not the 10% accent): `#2dd4a7` (`--color-success`) for the
"bot online" status dot and freshness-good state; `#ffb020` (`--color-warning`, shared value with
`--accent-reviews`) for "pending" badges/staleness warnings.

**Locked brand constraint (MANIFEST.md):** Nocturna graffiti-red `#e11d2e` (`--color-brand`) is
reserved for the logo mark only — never used for buttons, links, accents, or any interactive
chrome in the dashboard shell.

Phase 3 scope = shell chrome, Overview, 5 stub-module bodies, in-shell 403 page, and 2 new
mapping fields on the existing Settings page. No CRUD/approve/delete flows ship this phase (those
are Phases 6-9) — copy below covers only what Phase 3 actually renders.

| Element | Copy |
|---------|------|
| Primary CTA | "Guardar ajustes · Save settings" — the existing settings-page save button is reused unchanged for the new `manager_roles`/`editor_roles` fields; Phase 3 introduces no new primary CTA of its own. |
| Empty state heading (module stub) | "Próximamente · Coming soon" |
| Empty state body (module stub) | "Este módulo llega en una fase futura del dashboard. Vuelve pronto. · This module ships in a later dashboard phase. Check back soon." |
| Empty state heading (Overview recent activity, no rows yet) | "Sin actividad reciente · No recent activity" |
| Empty state body (Overview recent activity) | "Aparecerá aquí en cuanto el bot registre un evento. · It will appear here once the bot logs an event." |
| Error state (in-shell 403, D-16) | "Esta sección requiere acceso de {tier} · This section needs {tier} access" — body: "Contacta al propietario si crees que esto es un error. · Contact the owner if you believe this is a mistake." Bilingual, styled `forbidden.html`, distinct from `login.html`'s editor-only 403 copy (RESEARCH.md Pitfall 2). |
| Destructive confirmation | None this phase. No delete/remove action ships in Phase 3; the role→tier mapping edit is a normal validate-then-write save (existing owner-gated `/admin/settings` POST), not a destructive action requiring a separate confirm dialog. |

**Out-of-scope copy note:** Do NOT render an on/off module toggle switch (the sketch's `.mod-hdr`

kill-switches as an anti-feature ("those flows have no designed off state"). Phase 3's module
header uses the sketch's `.mod-hdr` visual shape (icon, title, accent border) MINUS the toggle
switch and MINUS the on/off badge.

## Phase 3 Component Notes

Supplementary detail beyond the template fields, for planner/executor use (source: sketch 001
variant A + CONTEXT.md D-01–D-16).

### Sidebar (`_sidebar.html`)
- All 7 sections always render, in fixed order: Overview, Gallery, Reviews, Reminders, Jinxxy
  Store, Meetings, Settings (SHELL-01). No per-tier trimming of the nav list (D-14).
- Each nav item: icon glyph + label + left-border in that section's accent color when active/on
  (`.nav-item.on`, `border-left: 3px solid var(--acc)`).
- Ungranted section (server-computed from the `roles` dict, never client JS): append a 🔒 glyph
  after the label, same visual weight as the sketch's `.cnt`/`.dot` slot (right-aligned via
  `margin-left: auto`). Locked items still link to the real route — the forbidden page is the
  graceful dead end (D-16), not a disabled/no-op link.
- No pending-count badges and no on/off status dots in Phase 3's nav (those sketch affordances —
  `.cnt`, `.dot` — belong to Gallery/Reviews queue counts and module on/off state; queue counts
  arrive with Phase 7, and on/off dots are the out-of-scope toggle above). Phase 3's nav shows
  only label + accent border + optional lock glyph.
- Editors-only session: every section shows locked except a distinct "Editor" link at the bottom
  of the nav pointing to the existing `/editor` route (D-15) — not one of the 7 accent-colored
  sections, no lock icon (it's their one granted destination).
- Footer status chip (`.foot`): reuse sketch shape (colored dot + "Bot en línea · vX" text) fed by
  the Phase 3 heartbeat table — dot color `--color-success` when heartbeat is fresh, otherwise
  `--color-text-faint` (no red/danger state needed here; absence of green is sufficient signal).

### Module header (`.mod-hdr`, used by all 5 stub pages)
- Icon + title + description line + left accent border in the module's color — per sketch, MINUS
  the toggle switch and on/off badge (see Copywriting Contract out-of-scope note).

### Overview page
- **Primary visual anchor: the bot-status stat tile** (leftmost, `--color-success`/`--color-danger`
  accent depending on heartbeat freshness) — it answers the single most important question on this
  screen ("is the bot up?") and should draw the eye first. The recent-activity table is the
  secondary focal point below it; the remaining stat tiles (last Jinxxy sync, etc.) are tertiary,
  same visual weight as each other.
- Stat tiles (`.stat`, 4-column grid, `--space-4` gap): Bot status (online/offline + uptime),
  pending-count-shaped tiles are NOT populated with real queue data yet (Phases 6-9 own those
  counts) — Phase 3 shows what D-09/D-10/D-11 plumbing actually provides: bot status, last Jinxxy
  sync time, and a placeholder/derived tile is acceptable only if backed by real heartbeat/sync
  data; do not fabricate a number with no data source.
- Recent activity: table of last ~10 `activity_log` rows (D-11), each row icon + message + relative
  timestamp, matching the sketch's activity table row shape exactly.
- No "Acciones rápidas" quick-action block — that is sketch variant C, explicitly deferred
  (FUT-02, Deferred Items in STATE.md). Phase 3 renders variant A's Overview only.

### Forbidden page (`forbidden.html`)
- New template, in-shell where feasible (wrapped in the same base layout so the sidebar is still
  visible, reinforcing "you can see the section exists, you just can't use it"), NOT a reuse of
  `login.html`. Bilingual copy per Copywriting Contract above. HTTP 403.
