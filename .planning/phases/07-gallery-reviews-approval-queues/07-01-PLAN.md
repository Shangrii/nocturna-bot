<objective>
Build the bot→app read side of Phase 7: a push-cache that makes the Discord-only
approval queue visible to the credential-less FastAPI app (GAL-01 / REV-01 read side).
A new bot cog scans the photo and reviews channels on a ~45s loop, classifies every
message via the cogs' OWN exported pure helpers (never duplicated logic), resolves the
poster/author display name from the live message, and UPSERTs rows into two new shared
sqlite tables. The app process (later plans) only ever SELECTs these tables.

Purpose: pending state lives ONLY as Discord reaction state (a ✅ prompt without a 🟢
marker) — the app has no Discord credentials and cannot read it. The push cache is the
sole viable route (D-01, locked).
Output: `core/db.py` queue helpers, `cogs/gallery_reviews_cache.py`, bot registration,
and the RED-first test file this phase's cache behavior is sampled against.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord gateway → bot process | Message content/attachments/author are attacker-influenceable input (any guild member can post) |
| Bot process → shared sqlite | The cache cog is the sole writer of gallery_queue/reviews_queue; the app only reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | reviews_queue anonymity | mitigate | The cog MUST classify via `_review_author_and_text`; when it returns `(None, text)` the row is written is_anonymous=1 / author=None — `message.author.display_name` is never written for an anonymous review (D-13). Test asserts this. |
| T-07-02 | Tampering | core/db.py queue helpers | mitigate | Every new helper uses `?` placeholders exclusively (parameterized SQL, ASVS V5) — no value is string-interpolated into SQL. |
| T-07-03 | Denial of Service | recurring channel scan | mitigate | Bounded `channel.history(limit=300)` per tick (Pitfall 3) prevents unbounded, ever-growing Discord REST cost as history accumulates. |
| T-07-04 | Tampering | shared-sqlite writer contention | mitigate | Writes go through `core/db.py::_get_conn()` (WAL + busy_timeout=8000, INFRA-02) — the cog never opens a raw connection bypassing the hardening (Pitfall 4). |
| T-07-SC | Tampering | package installs | accept | No new npm/PyPI package is added this phase (RESEARCH Package Legitimacy Audit: not applicable) — no supply-chain surface introduced. |
