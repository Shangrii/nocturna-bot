(`app/routers/gallery.py`, `app/routers/reviews.py`) that read the push cache for the

(written by 07-01) and enqueue a typed action (dispatched by 07-02). These routes are the
single seam between the browser and both mechanisms.

404-on-stale, JSON refresh).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → FastAPI route | Manager-authenticated but the message_id path/body param is user-supplied |
| FastAPI route → action_queue | The route is the only place a panel click becomes a queued Discord write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-09 | Elevation of Privilege | every gallery/reviews route | mitigate | Every route (page, /queue, approve, remove) depends on `require_manager` (owner OR Manager); a non-Manager gets 403 via TierForbidden. Integration test asserts the 403 on both a POST and the /queue route. |
| T-07-10 | Tampering (input validation) | message_id param | mitigate | `message_id: int` path typing (422 on non-int, ASVS V5); a message_id not present in the cache 404s before enqueue, so no arbitrary/stale id becomes a queued action. |
| T-07-11 | Tampering (kind drift) | _ALLOWED_KINDS | mitigate | The four new kinds are added to `_ALLOWED_KINDS` as a matched pair with the dispatcher (Pitfall 5) so the generic `/api/actions` path cannot smuggle an unknown kind nor reject a valid one. |
| T-07-12 | Information Disclosure | /queue + template context | mitigate | The GET routes pass ONLY cache rows (which already exclude anonymous reviewers' identities per 07-01/D-13) — the app never has access to a de-anonymizing field. Reviews /queue test asserts no real name for an anon row. |
| T-07-SC | Tampering | package installs | accept | No new package this phase. |
</threat_model>
