<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server-rendered template → browser | Cache row values (caption, poster, review text) are attacker-influenceable Discord content rendered into HTML |
| Browser Alpine → POST enqueue routes | Per-card actions post a message_id back to the Manager-gated routes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-13 | Information Disclosure | review author binding | mitigate | The card renders `is_anonymous`→"Anónimo" and only ever reads the author field the cache wrote (None for anon rows) — no template path can surface a real identity (D-13). Render test asserts this. |
| T-07-14 | Tampering (XSS via Discord content) | caption/review text rendering | mitigate | Jinja2 autoescaping is on by default; caption/poster/review text are rendered as escaped text (no `| safe`), so a crafted caption cannot inject markup. |
| T-07-15 | Spoofing (destructive misclick) | Remove action | mitigate | Remove is gated behind the `.confirm-modal` (D-10); Approve is one-click but reversible by Remove — no un-guarded destructive path. |
| T-07-16 | Tampering (double-submit) | lightbox action source | mitigate | The lightbox is view-only (no approve/remove inside) — a single action source (the card) prevents a second click-path double-submitting against the queue. |
| T-07-SC | Tampering | package installs | accept | No new package this phase (Alpine.js already vendored). |
</threat_model>
