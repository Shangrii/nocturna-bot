<objective>
Add a `require_owner` FastAPI dependency to `app/deps.py` that narrows the existing editor
session to the single configured owner (`config.DISCORD_USER_ID`), returning 403 for anyone else
and **failing closed** when the owner id is unset/`0` (PANEL-01, D-10). This is the phase's primary
security surface — build it test-first.

Purpose: Gate the entire `/admin/settings` surface. The owner-gate `0`-default trap and the
str-vs-int session/config type mismatch are the two documented ways this goes wrong; both get a
dedicated failing test before the implementation.
Output: `require_owner` in `app/deps.py` + `_OWNER_FORBIDDEN_COPY`, unit-tested in `tests/test_app_auth.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser session → require_owner | The session cookie asserts an identity; require_owner must trust ONLY the signed session, never the body |
| config → authorization | DISCORD_USER_ID's `0` default doubles as "unset"; must never authorize |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Elevation of Privilege | require_owner fail-open on `0` default | mitigate | Explicit `if not owner_id: raise 403` guard BEFORE any comparison (Pitfall 1). Dedicated test: owner unset (0) + any session → 403. |
| T-02-04 | Elevation of Privilege | str(session) vs int(config) type confusion | mitigate | Normalize both operands with str() before comparing (Pitfall 4). Dedicated test: int owner 555 + str session "555" → admitted. |
| T-02-05 | Spoofing / IDOR | identity sourced from request body | mitigate | Read identity from `request.session` only, mirroring require_editor's D-08 choke point. Acceptance criterion forbids body/query access in the function body. |
</threat_model>
