<objective>
Close the two FAILED Phase 2 roadmap success criteria (SC2/PANEL-02 and SC3/PANEL-03),
both documented in 02-VERIFICATION.md and both rooted in a single defect surface:
`core/settings.py::all_for_ui()` emits the editable payload with (a) snowflake/role_list
values as raw Python `int`/`list[int]`, and (b) the CONF-03 fallback already resolved.

- CR-01 (gap 1): Jinja `tojson` renders a bare `int` (e.g. `1416329356426481717`) as a bare
  JS number literal; the browser parses it as an IEEE-754 double, silently rounding every
  17-20 digit Discord snowflake before it is displayed. One "Save" with zero edits then
  persists the corrupted-but-still-numeric ID (it passes `str(value).isdigit()`), rewriting
  every channel/forum/role ID in the store.
- CR-02 (gap 2): `all_for_ui()` sources `value` via `get()`, which resolves the empty-list →
  `GALLERY_STAFF_ROLE_IDS` fallback. The client always posts the full flattened form, so an
  unmodified save bakes the resolved value into REVIEWS/REMINDERS/JINXXY_STAFF_ROLE_IDS,
  permanently destroying the CONF-03 read-time cascade (`test_staff_role_fallback_to_gallery`).

Both are fixed together, server-side, in `all_for_ui()`: expose the RAW stored value
(new `_get_raw()` helper that skips the `fallback_key` branch) and serialize snowflake →
`str`, role_list → comma-joined `str`. The existing validators already accept digit strings
and comma-separated strings, and `settings.html`'s text inputs already bind strings, so the
POST round-trip and the template need no further change: once `tojson` emits QUOTED strings,
the JS payload can no longer lose precision, and once the payload carries the raw (empty)
staff-role list, an unchanged save re-stores empty and the fallback keeps cascading at read time.

Purpose: Make the panel data-integrity-preserving end-to-end, restoring the phase's stated
Core Value ("without exposing secrets or letting a bad value break a cog") and not breaking
Phase 1's CONF-03.
Output: `_get_raw()` + string serialization in `core/settings.py::all_for_ui()`; unit and
integration regression tests that fail on the current HEAD and pass after the fix.
</objective>
