<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server render → browser | groups payload is serialized into the page; must carry only allowlisted tunables |
| client JS → server | save() posts owner-edited values; server (02-04) is the validation authority, not this JS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Information Disclosure | settings.html render | mitigate | Template iterates ONLY the `groups` payload (allowlisted 19 tunables from all_for_ui). Render smoke test asserts no BOT_TOKEN/GITHUB_PAT/JINXXY_API_KEY/SESSION_SECRET/DB_PATH in output. |
| T-02-07 | Tampering | client-side validation as authority | accept | Client `min`/`max`/`pattern` attributes are UX hints only; the server's validate_only/set is the sole authority (02-04). No security decision is made in JS. |
| T-02-08 | Cross-site scripting via x-data | settings.html hydrate | mitigate | Use the single-quoted `x-data='...{{ groups|tojson }}...'` idiom (Jinja escapes `'`), exactly as editor.html documents — a double-quoted host attribute would break on the JSON's first `"`. |
</threat_model>
