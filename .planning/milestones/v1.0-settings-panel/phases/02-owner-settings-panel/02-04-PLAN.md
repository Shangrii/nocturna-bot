<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → GET/POST /admin/settings | untrusted request; require_owner gates identity, settings.set gates values |
| POST body → settings store | owner-supplied key/value pairs cross into the shared sqlite the bot reads |
| error/success payload → browser | must never echo a secret or a structural value |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Elevation of Privilege | route access control | mitigate | Both routes use `Depends(require_owner)` (02-02, fail-closed). Test: non-owner → 403 with no data. |
| T-02-09 | Tampering | partial write on multi-field POST | mitigate | Two-pass validate-all-then-write-all; validate_only never writes. Regression test: 1 valid + 1 invalid → 0 writes (Pitfall 2). |
| T-02-10 | Tampering / Elevation | arbitrary-key write to settings table | mitigate | All writes go through settings.set, whose `_SCHEMA` allowlist rejects unknown keys; no raw SQL in app/main.py. |
| T-02-11 | Spoofing / IDOR | identity from request body | mitigate | Identity is `ident` from require_owner (session only); the body supplies only WHAT changes, never WHO asks. |
| T-02-12 | Tampering (CSRF) | state-changing POST | accept | SameSite=Lax session cookie + session-only identity, the existing accepted mitigation for /editor/save (RESEARCH Don't Hand-Roll). No new CSRF token. |
| T-02-13 | Information Disclosure | GET body / POST error payload | mitigate | GET renders only all_for_ui() (allowlisted); error map carries SettingRejected.reason (schema copy), never a secret. Test asserts secret-absence in GET body. |
| T-02-SC | Tampering (supply chain) | package installs | accept | Zero new packages this phase. |
</threat_model>
